# 文档总览

本项目文档按三个层面组织，各层独立演进：

| 层面 | 目录 | 内容 |
|------|------|------|
| 软件平台 | [01-软件平台/](./01-软件平台/) | 用户使用的 Web 设计平台：需求、产品规格、技术待决事项 |
| 印刷制作 | [02-印刷制作/](./02-印刷制作/) | 纪念券的实体制作工艺：流程、材料、调墨、丝印、荧光 |
| 商务流程 | [03-商务流程/](./03-商务流程/) | 开店渠道、激活码售卖、引流转化（规划中） |

## 各层说明

**01-软件平台**：三份文档的演进关系——`raw-requirements.md`（需求起源，对话式记录）→ `bdd-spec.md`（当前产品规格，开发以此为准）→ `open-questions.md`（待决清单）。改动规格时同步检查 open-questions 中对应条目是否需要更新或关闭。

**02-印刷制作**：主文档 `制作流程.html`（浏览器打开），流程性内容集中维护不拆分。物料清单、打样记录、供应商信息等如积累成规模，可另立文档放本目录。

**03-商务流程**：暂无正式文档。已知议题（待补充）：开店渠道（淘宝/微信小店等）、激活码售卖方式（手动上架 vs 电商 API，起步倾向手动，见 01 层 open-questions Q1）、引流路径、定价与成本平衡。后续按议题分文件放本目录。

## 推荐阅读顺序

- **了解产品要做什么**：`01-软件平台/raw-requirements.md`（需求起源）→ `01-软件平台/bdd-spec.md`（当前规格）
- **了解实体产品怎么做**：`02-印刷制作/制作流程.html`（浏览器打开）
- **排查悬而未决的决策**：`01-软件平台/open-questions.md`
